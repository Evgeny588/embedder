# Text Embedding Pipeline

Инструмент для преобразования текстовых документов в векторные
представления (эмбеддинги) с использованием архитектуры Sentence
Transformers. Система выполняет семантическую сегментацию текста,
кодирование фрагментов и их агрегацию в единый вектор фиксированной
размерности.

## 1. Системные требования

-   Python 3.9 или выше
-   Менеджер пакетов pip
-   Файл `requirements.py`
-   Опционально: CUDA-совместимый GPU и соответствующие драйверы

## 2. Установка и настройка

### 2.1. Установка зависимостей

Выполните установку необходимых библиотек. Запустите скрипт:

    pip install -r requirements.py


### 2.2. Настройка конфигурации (.env)

1.  Создайте файл `.env` в корневой директории проекта.
2.  Добавьте переменную `CHECKPOINT`, указывающую на идентификатор
    модели в Hugging Face Hub или локальный путь к весам.

Пример содержимого `.env`:

    CHECKPOINT=sentence-transformers/all-MiniLM-L6-v2

## 3. Запуск пайплайна

Скрипт запускается через командную строку с обязательными аргументами.

    python main.py --filename <имя_файла> --device <устройство>

### Параметры запуска

  | Аргумент |       Тип |  Значение по умолчанию |  Описание |
  |:---|:---|:---|:---|
  |`--filename` |  str  | none        |            Имя файла в папке `raw_text` (обязательно)|
  |`--device`    | str  | cpu            |         Вычислительное устройство: `cpu` или `cuda`|

**Пример запуска на CPU:**

    python main.py --filename document.txt --device cpu

## 4. Логика работы

Пайплайн использует гибридный подход к агрегации векторов (Pooling), что
обеспечивает более устойчивое представление текста по сравнению со
стандартным усреднением.

### 4.1. Сегментация

Используется `SentenceChunker`. Текст разбивается на чанки размером 450
токенов с перекрытием (overlap) 50 токенов. Перекрытие необходимо для
сохранения контекстной связности на границах фрагментов.

### 4.2. Инференс

Каждый чанк пропускается через модель. К тексту каждого чанка
добавляется префикс `passage:`, так как модель оптимизирована для
асимметричного поиска (passage vs query).

### 4.3. Агрегация

Финальный вектор рассчитывается как среднее
арифметическое двух стратегий пулинга


## 5. Логирование и вывод

### 5.1. Логи

-   **Консоль:** Выводятся сообщения уровня `INFO` и выше. Ошибки
    (`ERROR`) выделяются красным цветом.
-   **Файл:** Полные логи (включая `DEBUG`) пишутся в `logs/logs.log`.
    Файл ротируется при достижении 5 МБ.

### 5.2. Результат

Финальный вектор сохраняется в файл `outputs/out.txt` в формате строки
(список чисел с плавающей точкой). Файл перезаписывается при каждом
успешном запуске.

**Важно:** При первом запуске требуется доступ к интернету для загрузки
весов модели. Дальнейшие запуски используют кэш из папки `model_cache/`
и работают офлайн.
